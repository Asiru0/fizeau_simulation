# Fizeau 干涉阵成像仿真

这是一个面向稀疏 Fizeau 干涉阵的 Streamlit 交互仿真应用，可以从子孔径布局出发，计算单色或宽谱 OTF/MTF，生成带噪观测，并使用单姿态或多姿态 Wiener 方法复原图像。

默认参数为 80 mm 子孔径、28 m 有效焦距、450--650 nm 光谱，以及 B1、B2、B3 三组矩形阵列构型。

## 主要功能

- 自定义子孔径数量、中心坐标和统一孔径口径。
- 输入焦距、探测器尺寸及宽谱相对强度。
- 计算阵列布局、宽谱 OTF、MTF 和几何视场。
- 生成归一化分辨率板的 OTF 退化图像。
- 加入探测器的高斯噪声。
- 使用 Wiener 滤波实现单姿态图像复原。
- 编辑多个阵列构型并为每个构型生成旋转姿态。
- 使用等权、噪声方差倒数或手动权重进行多姿态融合。
- 支持标准多姿态 Wiener 和软自适应 Wiener。
- 输出 PSNR、前景平衡 PSNR、SSIM 和梯度相似度。

## 环境安装

建议使用 Python 3.10 或更高版本，并在项目根目录创建虚拟环境：

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -r requirements.txt
```

主要依赖包括 NumPy、Matplotlib、Streamlit 和 pytest。

## 启动交互应用

```bash
source .venv/bin/activate
streamlit run app.py
```

默认访问地址：

```text
http://localhost:8501
```

应用顶部可以在“单姿态”和“多姿态合成”之间切换。所有计算参数使用文本输入；只有点击运行按钮后才会执行仿真。图像尺寸目前固定为 `256 x 256`。

## 宽谱输入

光谱输入框每行包含波长和相对强度：

```text
450 0.2
500 0.2
550 0.2
600 0.2
650 0.2
```

波长单位为 nm。程序会校验所有波长和强度，并将相对强度再次归一化：

```text
normalized_weight_i = intensity_i / sum(intensity)
```

随后分别计算每个波长的单色 OTF，并按归一化权重合成为宽谱 OTF。只有一行光谱数据时，计算自然退化为单色仿真。空行会被忽略，负强度和全零光谱会被拒绝。

## 单姿态仿真

单姿态模式包含以下输入：

- 子孔径中心位置，单位为 mm。
- 统一子孔径口径，单位为 mm。
- 有效焦距和探测器边长，单位为 mm。
- 宽谱波长及相对强度。
- 高斯噪声相对强度。
- Wiener 正则化系数。
- 可选随机种子；留空时每次运行生成新的噪声。

子孔径默认为 4 个，可以动态增加或删除。运行后显示：

- 按实际孔径大小绘制的阵列布局。
- 频率坐标单位为 cycles/mm 的 MTF。
- 归一化目标、OTF 退化图、带噪观测和 Wiener 复原图。
- 几何视场、底片比例尺及图像质量指标。

## 多姿态合成

### 阵列构型

多姿态模式默认提供 B1、B2、B3 三个矩形构型：

| 构型 | 矩形对角线 | 默认旋转角 |
| --- | ---: | --- |
| B1 | 310 mm | 0, 45, 90, 135 deg |
| B2 | 745.556 mm | 0, 45, 90, 135 deg |
| B3 | 800 mm | 0--168.75 deg，步长 11.25 deg |

构型可以增加、删除和重命名，每个构型中的子孔径也可以独立增删。重命名构型时，姿态计划中的引用会同步更新。

### 规则姿态生成

规则生成器接受：

- 阵列构型。
- 起始角度。
- 角度步长。
- 姿态数量。
- 名称前缀。

点击“追加到姿态计划”后，生成的姿态会附加到当前计划末尾，不会立即运行仿真。

### 姿态计划格式

姿态计划使用七列文本格式：

```text
# name array rotation_deg weight noise_sigma seed enabled
B1_0   B1 0    1.0 - - 1
B1_45  B1 45   1.0 - - 1
B2_0   B2 0    0.8 0.03 2001 1
B3_off B3 11.25 1.0 - - 0
```

| 列 | 含义 |
| --- | --- |
| `name` | 姿态唯一名称 |
| `array` | 引用的阵列构型名称 |
| `rotation_deg` | 相对基础构型的逆时针旋转角，单位为 deg |
| `weight` | 手动融合模式下的相对权重 |
| `noise_sigma` | 该姿态的噪声强度；`-` 表示继承公共值 |
| `seed` | 该姿态的随机种子；`-` 表示从基础种子派生 |
| `enabled` | `1` 参与融合，`0` 忽略 |

支持空行和 `#` 注释。“校验姿态计划”只进行解析和归一化检查，不计算 OTF。默认计划包含 24 个启用姿态。

当填写基础随机种子时，继承种子的姿态使用：

```text
pose_seed = base_seed + pose_index
```

基础随机种子留空时，各姿态使用非固定随机噪声。

### 融合权重

多姿态模式支持三种权重方式：

- 等权：所有启用姿态权重相同。
- 噪声方差倒数：权重与 `1 / sigma^2` 成正比。
- 手动权重：使用姿态计划中的 `weight`。

所有实际融合权重都会归一化。参与融合的姿态权重必须大于 0。

### 融合方法

标准多姿态 Wiener 使用加权频域最小二乘形式：

```text
X_hat = sum(w_i * conj(H_i) * Y_i)
        / (sum(w_i * |H_i|^2) + K)
```

软自适应 Wiener 在复合 MTF 上增加软频率权重和随频率变化的正则化，适合降低低覆盖频率区域的噪声放大。相关输入包括：

- 融合正则化系数。
- MTF 阈值。
- 软过渡宽度。

多姿态运行结果包括复合 MTF、第一帧观测、融合复原图像和质量指标。

## 当前模型假设

- 非相干成像。
- 子孔径为相同口径的圆形孔径。
- 阵列理想共相，不包含活塞、倾斜或波前误差。
- 所有姿态观察同一个归一化目标。
- 宽谱成像由离散单色 OTF 的强度加权和近似。
- 探测器噪声使用独立零均值高斯噪声近似。
- 当前交互应用不模拟曝光时间、饱和、暗电流、读出偏置和姿态间配准误差。
- 图像评价使用已知无噪目标，仅适用于合成数据实验。

后续可继续加入实测光谱响应、Poisson 光子噪声、探测器非理想效应、活塞相位误差、离轴视场变化、姿态配准误差以及真实数据下的参数选择方法。
